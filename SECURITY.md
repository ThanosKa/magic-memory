## Reporting a Vulnerability

- Email: `security@magic-memory.app`
- Include a short description, reproduction steps, and potential impact.
- Please do not open public issues for security reports.

We aim to acknowledge new reports within 3 business days and provide an initial assessment within 7 business days.

## Handling Process

1. Triage and validate the report.
2. Develop and test a fix.
3. Coordinate a release and disclosure timeline with the reporter.
4. Publish security notes in the release changelog.

## Scope

This policy covers the application code in this repository and the infrastructure defined here. Third-party services (Clerk, Supabase, Stripe, Replicate, Upstash) should be reported directly to their respective security channels.
